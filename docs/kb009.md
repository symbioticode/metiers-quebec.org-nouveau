# kb009 — Collision guard : résolution par algorithme node-centric

## Problème (résolu)

L'algorithme slug-par-slug avec guard anti-collision présentait une asymétrie :
les slugs sans clé exacte dans prefix_map (ex: `vendeur`) étaient contaminés
par les sous-concepts de leurs frères (ex: `vendeur_autos_*`), car le guard
ne s'appliquait que aux slugs avec exact match.

## Solution : longest-prefix-wins (node-centric)

Remplacement complet de la logique. Au lieu de chercher les noeuds pour chaque slug,
on assigne chaque noeud au slug le plus long qui le couvre :

```
Pour chaque noeud du graphe (key = ID sans préfixe "metier_"):
  Trier corpus_slugs par longueur décroissante
  Trouver le PREMIER slug s tel que key == s OU key.startswith(s + "_")
  Attribuer ce noeud exclusivement à s
```

`vendeur_autos_representant_en_vente_automobile` est attribué à `vendeur_autos`
(14 chars) plutôt qu'à `vendeur` (7 chars) car `vendeur_autos` est plus long.

## Résultat avant/après

| Critère | Avant (slug-par-slug) | Après (node-centric) |
|---------|----------------------|----------------------|
| `vendeur.subnodes` | 7 dont 4 vendeur_autos/quincaillerie | **1** : `metier_vendeur_index_vendeur` |
| `vendeur_autos` | manquant ou fusionné | **Existe** : `metier_vendeur_autos_representant...` |
| `vendeur_quincaillerie` | manquant ou fusionné | **Existe** : `metier_vendeur_quincaillerie_commis...` |
| `peintre.graph_node_id` | `metier_peintre_peintre_en_batiment` | **`metier_peintre_peintre_en_batiment`** (même) |
| `peintre_indl` | fusionné dans peintre | **Entrée séparée** : `metier_peintre_indl` |
| Slugs matchés | 416/418 | **416/418** (stable) |
| Noeuds non-assignés | — | **25** (hors corpus) |

## Limite restante

25 noeuds du graphe n'ont pas de slug corpus correspondant (clés comme
`metier_agent_socio_sanl`, `metier_controleur_aerien_aerien_controleur`, etc.).
Ces noeuds sont dans le graphe mais pas dans le corpus scrapé — ce ne sont
pas des bugs d'assignation, ce sont des métiers du graphe sans fiche détaillée.
