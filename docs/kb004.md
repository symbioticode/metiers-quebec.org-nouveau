# KB004 — Analyse de connaissances avec Graphify

## Objectif

Transformer le site metiers-quebec.org (420 métiers, 30 secteurs, 457 pages HTML) en un **graphe de connaissances** queryable pour alimenter un chatbot RAG.

## Outil utilisé

- **Graphify** v0.9.23 — https://github.com/Graphify-Labs/graphify
- License MIT, 93k+ étoiles GitHub
- Transforme code/docs/HTML/PDF en graphe de connaissances (nœuds + arêtes + communautés)

## Installation

```bash
# Installer graphify
uv tool install graphifyy

# Extra pour DeepSeek (API OpenAI-compatible)
uv tool install "graphifyy[openai]" --force

# Installer le skill OpenCode
graphify install --platform opencode
```

## Backend LLM utilisé

- **DeepSeek** (API OpenAI-compatible)
- Coût : ~$0.41 pour 452 fichiers HTML
- 172 chunks en cache, 280 re-extraits
- Tokens : 1.7M input / 618k output

## Problèmes rencontrés et solutions

### 1. Timeout de l'extraction (10 min insuffisant)

**Problème** : L'extraction de 452 fichiers HTML a timeout après 10 minutes (21/49 chunks traités).

**Solution** : Relancer avec un timeout de 30 minutes. L'extraction complète a pris ~25 minutes.

**Leçon** : Pour 400+ fichiers HTML, prévoir au moins 30 minutes d'extraction.

### 2. Package `openai` manquant pour DeepSeek

**Problème** : `graphify extract --backend deepseek` échoue avec "the 'openai' package is required".

**Solution** : `uv tool install "graphifyy[openai]" --force` — DeepSeek utilise l'API OpenAI-compatible.

**Leçon** : DeepSeek nécessite l'extra `openai` de graphifyy, pas un package séparé.

### 3. Dossier `dist/dist/` en doublon

**Problème** : Un dossier `dist/dist/` contenait une copie complète du site (457 fichiers en plus), portant le total à 914 fichiers.

**Solution** : `rm -rf dist/dist/` avant l'extraction.

**Leçon** : Vérifier l'absence de doublons dans le dossier cible avant extraction.

### 4. Nœuds en double (métiers listés dans plusieurs pages)

**Problème** : Graphify émet ~100 warnings du type "node X is minted by two different files". Un métier apparaît dans :
- `alpha/index.html` (index alphabétique)
- `secteur/XXX/index.html` (page secteur)
- `metier/XXX/index.html` (fiche métier)

**Solution** : Graphify déduplique automatiquement (20 nœuds fusionnés : 1 exact + 19 fuzzy). Les warnings sont informatifs.

**Leçon** : C'est normal pour un site avec navigation croisée. Le graphe reste cohérent.

### 5. Pas de nommage des communautés

**Problème** : `graphify cluster-only` affiche "Community N" sans noms significatifs.

**Solution** : Passer un backend LLM pour le label : `graphify cluster-only dist/ --backend deepseek`.

**Leçon** : Le nommage des communautés nécessite un appel LLM supplémentaire.

## Commandes utiles

```bash
# Interroger le graphe
graphify query "quels métiers liés à la santé?"
graphify explain "infirmier"
graphify path "infirmier" "informatique"

# Visualiser
xdg-open dist/graphify-out/graph.html

# Relancer l'extraction (depuis le cache)
graphify extract dist/ --backend deepseek

# Servir pour chatbot (MCP)
python -m graphify.serve dist/graphify-out/graph.json
```
