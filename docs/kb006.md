# kb006 — Choix du modèle d'embedding

## Décision

**Modèle retenu** : `BAAI/bge-small-en-v1.5`

| Propriété | Valeur |
|-----------|--------|
| Auteur | BAAI (Beijing Academy of AI) |
| Params | ~33M |
| Dimensions | 384 |
| Taille fichier | ~130 Mo (ONNX) |
| Licence | MIT |
| Framework | fastembed (ONNX Runtime) |
| PyTorch requis | Non |
| Multilingue | Oui (données d'entraînement incluent du français) |

## Pourquoi ce modèle

1. **Léger** — 33M params vs 110M+ pour bge-base. Pas de GPU nécessaire.
2. **ONNX natif** — fastembed utilise ONNX Runtime, pas besoin de PyTorch.
3. **Gratuit** — Licence MIT, pas de clé API.
4. **Multilingue** — bge-small a du multilingue dans ses données d'entraînement, fonctionne raisonnablement sur le français.
5. **Bon ratio qualité/taille** — 384 dimensions suffisent pour la recherche sémantique sur un corpus de 418 métiers.

## Alternatives évaluées

| Modèle | Params | Dim | PyTorch | Multilingue | Notes |
|--------|--------|-----|---------|-------------|-------|
| BAAI/bge-small-en-v1.5 | 33M | 384 | Non (fastembed) | Oui | **Retenu** |
| BAAI/bge-base-en-v1.5 | 110M | 768 | Oui | Oui | Trop lourd pour CPU |
| sentence-transformers/all-MiniLM-L6-v2 | 22M | 384 | Oui | Non | Anglais seulement |
| snowflake-arctic-embed-l | 335M | 1024 | Oui | Non | Trop lourd |
| text-embedding-3-small (OpenAI) | — | 1536 | — | Oui | Payant (~$0.02/1M tokens) |

## Coût

| Poste | Coût |
|-------|------|
| Modèle | Gratuit (MIT) |
| Inférence | Gratuit (CPU, pas de GPU) |
| Stockage embeddings | 81 Mo (7578 chunks × 384 floats × 4 bytes) |
| API externe | Aucune |
| **Total** | **0 $** |

Comparaison avec une API payante (OpenAI text-embedding-3-small) :
- 7578 chunks × ~500 mots/chunk = ~3.8M mots ≈ ~5M tokens
- Coût OpenAI : ~$0.10 (input) + stockage
- Notre solution : gratuit

## Performance mesurée

| Métrique | Valeur |
|----------|--------|
| Temps total embedding | ~43 min (CPU, batch 64) |
| Débit | ~2-3 chunks/seconde |
| Fichier sortie | 81 Mo (JSONL) |
| Reprise automatique | Oui (skip slugs déjà traités) |

## Limitations connues

1. **Anglais dominant** — Le modèle est principalement entraîné sur l'anglais. Le français fonctionne mais avec une qualité inférieure à un modèle multilingue dédié (comme multilingual-e5-small).
2. **384 dimensions** — Moins discriminant que 768 ou 1024 dimensions pour des requêtes très spécifiques.
3. **Pas de fine-tuning** — Le modèle n'est pas affiné sur le vocabulaire québécois des métiers.

## Usage dans le projet

```python
from fastembed import TextEmbedding

model = TextEmbedding("BAAI/bge-small-en-v1.5")
embeddings = list(model.embed(["texte à embedder"], batch_size=64))
# embeddings[0] → array de 384 floats
```

Script : `scripts/embed_corpus.py`
Sortie : `data/embeddings/chunks.jsonl`
