# KB001 — Violations d'accessibilité (axe-core) sur le site statique

## Contexte

Après génération du site statique (446 pages HTML), un audit automatisé avec axe-core a révélé des violations WCAG 2 AA sur les types de pages suivants : accueil, stats, secteurs, alpha, secteur.

L'objectif était d'atteindre 0 violation avant déploiement.

## Vérification

### Outils utilisés

| Outil | Version | Rôle |
|-------|---------|------|
| axe-core | 4.12.1 | Moteur d'audit WCAG |
| puppeteer-core | — | Lancement Chrome headless |
| @axe-core/puppeteer | — | Intégration axe-core + Puppeteer |
| Chromium (NixOS) | /run/current-system/sw/bin/chromium | Navigateur headless |
| htmlhint | 1.9.2 | Validation HTML basique |

### Commande d'audit

```bash
cd /tmp && node audit_stats.mjs
```

Script `audit_stats.mjs` :
```javascript
import puppeteer from 'puppeteer-core';
import { AxePuppeteer } from '@axe-core/puppeteer';

const browser = await puppeteer.launch({
  executablePath: '/run/current-system/sw/bin/chromium',
  headless: true,
  args: ['--no-sandbox', '--disable-gpu', '--disable-dev-shm-usage']
});
const page = await browser.newPage();
await page.goto('http://localhost:8080/stats/', { waitUntil: 'networkidle2', timeout: 30000 });
const results = await new AxePuppeteer(page).analyze();
// ... formatage des résultats
await browser.close();
```

### Résultats initiaux (avant correction)

**Accueil (`/`)** — 2 violations :
- `color-contrast` [serious] — 32 éléments (`.sector-card__count`, `.section__link`)
- `landmark-unique` [moderate] — 2 `<nav>` sans labels uniques

**Stats (`/stats/`)** — 1 violation :
- `landmark-unique` [moderate] — même problème

**Alpha (`/alpha/`)** — 2 violations :
- `color-contrast` [serious] — `.profession-item` (tous les liens de la liste)
- `landmark-unique` [moderate] — `<nav class="breadcrumb">` sans label

**Secteur (`/secteur/sante/`)** — 1 violation :
- `color-contrast` [serious] — `.profession-item`

## Confirmation

Les couleurs calculées par le navigateur ont été extraites pour vérifier le contraste :

```javascript
// Résultat pour .sector-card__count : rgb(148, 163, 184) sur fond blanc
// → contraste ~2.8:1 (WCAG AA exige 4.5:1 pour texte normal)
// Résultat pour .section__link : rgb(59, 130, 246) sur fond blanc
// → contraste ~3.5:1 (insuffisant)
```

## Réparation

### 1. `color-contrast` — .sector-card__count

**Avant :**
```css
--text-lighter: #94a3b8;  /* contraste 2.8:1 */
.sector-card__count { color: var(--text-lighter); }
```

**Après :**
```css
--text-lighter: #64748b;  /* contraste 4.7:1 ✓ */
```

### 2. `color-contrast` — .section__link

**Avant :**
```css
.section__link { font-size: 0.85rem; font-weight: 500; }
/* hérite color du parent → #3b82f6 → contraste 3.5:1 */
```

**Après :**
```css
.section__link { font-size: 0.85rem; font-weight: 500; color: var(--primary); }
/* #1e40af → contraste 8.6:1 ✓ */
```

### 3. `color-contrast` — .profession-item

**Avant :** aucune règle CSS → couleurs navigateur par défaut (insuffisantes)

**Après :**
```css
.profession-item {
  display: block;
  color: var(--text);          /* #1e293b → contraste 14.5:1 ✓ */
  padding: 10px 16px;
  border-bottom: 1px solid var(--border-light);
  text-decoration: none;
  transition: background 0.15s, color 0.15s;
}
.profession-item:hover {
  background: var(--bg-light);
  color: var(--primary);
  text-decoration: none;
}
```

### 4. `landmark-unique` — nav header + nav sidebar

**Avant :**
```html
<nav class="nav" id="nav">...</nav>
<aside class="sidebar"><nav class="sidebar__nav">...</nav></aside>
```

**Après :**
```html
<nav class="nav" id="nav" aria-label="Navigation principale">...</nav>
<aside class="sidebar"><nav class="sidebar__nav" aria-label="Navigation latérale">...</nav></aside>
```

### 5. `landmark-unique` — nav breadcrumb

**Avant :**
```html
<nav class="breadcrumb">...</nav>
```

**Après :**
```html
<nav class="breadcrumb" aria-label="Fil d'Ariane">...</nav>
```

## Mesures de non-régression

1. **Audit automatisé** — relancer `audit_stats.mjs` ou script équivalent après toute modification CSS/HTML
2. **Test multi-pages** — vérifier au minimum : `/`, `/stats/`, `/alpha/`, `/secteur/sante/` (types différents)
3. **Vérification du contraste** — tout ajout de couleur doit être validé avec un outil comme [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) ou extrait via `getComputedStyle()` + calcul
4. **Règle `aria-label`** — tout élément `<nav>` doit avoir un `aria-label` unique

## Do

- Utiliser des variables CSS (`--text`, `--primary`) pour les couleurs de texte et maintenir la cohérence
- Vérifier le ratio de contraste WCAG AA (4.5:1 texte normal, 3:1 texte large) avant d'ajouter une couleur
- Ajouter `aria-label` sur tous les `<nav>` pour distinguer les land landmarks
- Exécuter axe-core sur un échantillon représentatif de pages (pas juste la page d'accueil)
- Garder un script d'audit reproductible dans le projet

## Don't

- Ne pas utiliser de couleurs trop claires (`#94a3b8` et plus clair) pour du texte sur fond blanc
- Ne pas ajouter de `<nav>` sans `aria-label` (violation `landmark-unique`)
- Ne pas ignorer les violations `serious` même si le site « a l'air bien »
- Ne pas se fier uniquement à l'audit sur une seule page — les classes CSS partagées propagent les erreurs partout
- Ne pas modifier `--text-lighter` ou `--text-light` sans vérifier l'impact sur tous les utilisateurs de ces variables
