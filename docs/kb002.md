# KB002 — Dette technique : Fiches emploi incomplètes et mapping manquant

## Contexte

Les fiches emploi du site généré (`/metier/*/`) sont structurellement différentes du site de référence (`localhost:8081`). Le formatage est « brut » : sections sans icônes, pas de cartes meta, contenu mal structuré, texte avec sauts de ligne, description absente pour beaucoup de métiers.

Le problème est double :
1. **Mapping incomplet** — les clés de sections dans les données JSON ne correspondent pas aux clés attendues par `generate.py`
2. **Données brutes** — le scraper capture le texte tel quel, sans nettoyage (newlines, métadonnées mélangées au contenu)

**Problème racine** : le scraper stocke les clés en MAJUSCULES avec des `\n` (ex: `TÂCHES\nET\nRESPONSABILITÉS`) alors que `generate.py` cherche des clés en minuscules simples (`taches`).

### Mapping manquant identifié

1. **Clés avec newlines** : `TÂCHES\nET\nRESPONSABILITÉS` → devrait être `taches`
2. **Clés alternatives** : `MILIEUX\nDE\nTRAVAIL` vs `MILIEU\nDE\nTRAVAIL` → devrait mapper vers `milieu`
3. **Clés avec apostrophes** : `EXIGENCES\nD'ADMISSION` → devrait mapper vers `admission`
4. **Clés manquantes** : `QUALITÉS\nET\nAPTITUDES\nREQUISES` existe mais n'est presque jamais remplie

## Dette Technique

### 1. Mapping des clés de sections

**Problème** : Le scraper utilise `part.upper().strip()` comme clé, ce qui crée des clés comme `TÂCHES\nET\nRESPONSABILITÉS`. Le generateur cherche des clés minuscules simples.

**Solution nécessaire** : Normaliser les clés au scraper OU ajouter un dictionnaire de mapping complet dans `generate.py`.

```python
# Mapping requis (clé scraper → clé generateur)
SECTION_KEY_MAP = {
    "TÂCHES\nET\nRESPONSABILITÉS": "taches",
    "MILIEU\nDE\nTRAVAIL": "milieu",
    "MILIEUX\nDE\nTRAVAIL": "milieu",
    "QUALITÉS\nET\nAPTITUDES\nREQUISES": "qualites",
    "EXIGENCES\nDU\nMARCHÉ\nDU\nTRAVAIL": "marche",
    "EXIGENCES\nD'ADMISSION": "admission",
    "EXIGENCE\nD'ADMISSION": "admission",
    "FORMATION\nREQUISE": "formation",
    "FORMATION\nREQUIS": "formation",
    "DONNÉES\nSALARIALES": "salaires",
    "DONNÉES\nSALARIALE": "salaires",
    "DONNÉE\nSALARIALE": "salaires",
    "STATISTIQUES\nDE\nPLACEMENT": "placement",
    "PERSPECTIVES\nD'EMPLOI": "perspectives",
    "DESCRIPTION": "description",
    "NIVEAU\nD'ÉTUDES": "niveau",
}
```

### 2. Nettoyage du contenu texte

**Problème** : Le texte contient des sauts de ligne entre chaque mot (`dans\nles\nhôpitaux`), des métadonnées mélangées au contenu (SECTEUR:..., NIVEAUX D'ÉTUDES:...).

**Solution nécessaire** :
- Joindre les lignes avec espaces avant formatage
- Extraire les métadonnées du bloc `intro` pour les cartes `prof-meta`
- Nettoyer les caractères superflus (points-virgules en début de paragraphe, etc.)

### 3. Structure des fiches emploi

**Éléments manquants ou incomplets** :

| État | Élément | Description |
|------|---------|-------------|
| ✅ Fait | `page-header__back` | Lien retour au secteur |
| ✅ Fait | `page-header__tag` | Tag secteur |
| ✅ Fait | `page-header__title` | Nom du métier |
| ✅ Fait | `page-header__meta` | Niveau d'études |
| ⚠️ Partiel | `prof-meta` cards | Secteur + Niveau OK, Salaire et Demande manquants |
| ⚠️ Partiel | `content-section` | Icônes SVG OK, mais contenu brut |
| ❌ Manquant | `prof-meta` card Salaire | Données salariales non extraites |
| ❌ Manquant | `prof-meta` card Demande | Pas de source de données |
| ❌ Manquant | Listes `<ul>/<li>` | Le contenu est du texte brut, pas structuré |

### 4. Règles de formatage

**Titre** :
- Capitaliser la première lettre de chaque mot (`title()`)
- Ex: `infirmier` → `Infirmier`, `comptable` → `Comptable`

**Breadcrumb** :
- Utiliser le nom capitalisé dans le dernier segment
- Séparateurs `/` avec span `.breadcrumb__sep`

**Prof-meta cards** :
- Nombre maximum : 4 cartes
- Ordre : Secteur → Niveau d'études → Salaire moyen → Demande
- Si pas de données, ne pas afficher la carte (pas de « Non disponible »)

**Content-sections** :
- Maximum 10 sections par fiche
- Chaque section doit avoir une icône SVG
- Le contenu doit être en paragraphes `<p>` et listes `<ul>/<li>`
- Maximum 500 caractères par paragraphe

### 5. Typographie et rédaction

**À supprimer du contenu** :
- « Information non disponible. » (quand la section est vide, ne pas afficher la section)
- Sauts de ligne entre les mots
- Points-virgules en début de paragraphe
-文本 en MAUSCULES sauf si c'est un acronyme
- Répétitions de mots (ex: « infirmier(ère) infirmier(ère) »)

**Règles de rédaction** :
- Utiliser le tutoiement (« tu ») si le site source l'utilise
- Garder le format « infirmier(ère) » pour les métiers genrés
- Phrases courtes (max 25 mots)
- Listes à puces pour les établissements, les tâches, les qualités

## Plan de résolution

1. Ajouter le dictionnaire `SECTION_KEY_MAP` complet dans `generate.py`
2. Appliquer le nettoyage texte (`" ".join(item.split())`) avant formatage
3. Masquer les sections vides au lieu d'afficher « Information non disponible. »

### Moyen terme (v0.2.0)
1. Extraire les données salariales des tableaux HTML
2. Ajouter une source de données pour la « Demande »
3. Structurer les listes en `<ul>/<li>` au lieu de texte brut
